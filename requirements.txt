RESEARCH APP V2 — TOP RESEARCHERS BY AREA (Streamlit)

O QUE MUDOU
- Engine selecionável:
  1) Auto: usa SerpAPI se houver SERPAPI_KEY; se falhar, faz fallback para Scholarly.
  2) Apenas SerpAPI.
  3) Apenas Scholarly.
- Proxies gratuitos (scholarly) opcionais.
- Pausas e backoff contra bloqueios do Scholar.
- Mensagens de erro mais claras.

COMO USAR
1) Instale dependências:
   pip install streamlit requests scholarly
2) (Opcional) Configure SERPAPI_KEY:
   export SERPAPI_KEY="sua_chave"
3) Rode:
   streamlit run research_app_v2.py

QUANDO USAR CADA ENGINE
- SerpAPI: produção/estabilidade, menos bloqueios e manutenção.
- Scholarly: ambiente de testes/estudos. Ative proxies gratuitos na UI.

ERROS COMUNS E SOLUÇÕES
- "Falha ao obter proxies": desmarque "Usar proxies" ou tente novamente (rede/provedores variam).
- "Import scholarly falhou": instale/atualize a lib (`pip install -U scholarly`).
- Nenhum resultado: tente termos mais amplos; reduza "Qtd. pesquisadores".
- Bloqueios do Scholar: intervalos maiores, menos requisições, use SerpAPI.

BOAS PRÁTICAS
- Limite de requisições por sessão.
- Use cache externo (ex.: salvar resultados em CSV/DB).
- Prefira SerpAPI para apps públicos/Streamlit Cloud.
